E-Commerce Product Catalog Hierarchy

Java Full Stack Capstone - Module C (PS-29)

1. Project Overview

This project implements a polymorphic E-Commerce Product Catalog. It solves the problem of extensive instanceof type checks in procedural code by introducing an abstract Product class and specific subtypes (Physical, Digital, Subscription).

The goal is to calculate varying deliverability and tax rates polymorphically.

2. Object-Oriented Architecture

The architecture leans heavily on Inheritance, Abstraction, and Polymorphism.

Abstraction & Inheritance

The Product class acts as an abstract base. Subclasses inherit shared properties like id, name, and basePrice, while defining their own specific properties (e.g., weightInKg for physical goods).

Polymorphism (The Hooks)

Each subtype overrides specific abstract methods:

  • calculateShippingCost(Region region): Meaningful for Physical, returns 0 for Digital and Subscriptions.
  • calculateTax(Region region): Applies different percentages based on product classification and region.
  • checkAvailability(): Maps to stock quantities, server status, or subscription activity flags depending on type.

4. Avoiding 'instanceof'

By using the Template Method pattern in Product.calculateTotalCost(), the catalog iterates purely on references of type Product. The JVM dynamically binds method calls to the specific subclass instance at runtime.

// Example of Polymorphism in CatalogService
for (Product product : catalog) {
    if (product.checkAvailability()) {
        System.out.println("Shipping: " + product.calculateShippingCost(region));
        System.out.println("Tax: " + product.calculateTax(region));
    }
}
